How the Agent Decides and Answers

Step 1: Identify the metric and time horizon in your question (guidance, quarter, full year, or scenario).

Step 2: Pull the public baseline and narrative from approved reports (financial + activity context and management commentary).

Step 3: Pull model-side actuals, budget, and forecast views from IBM Planning Analytics where available.

Step 4: Reconcile report numbers versus model state, quantify variance, and flag guidance or liquidity risk early.

Step 5: Return an executive-ready answer: direct conclusion, key drivers, and optional what-if sensitivity.

Discovery Tools

list_cubes

List all business cubes available in Planning Analytics. Returns cube names (system cubes excluded). Use this first to discover what data is available.

→ FinancialPerformance→ ProductionKPI→ CapexPlanning→ CashFlow

list_views

List all public named views defined for a given cube. Views are pre-built slices — useful before calling query_cube.

get_cube_dimensions

Return the ordered dimension list for a cube. Essential context before writing MDX or applying filters.

cube_name

get_dimension_elements

Return all elements (members) from a dimension hierarchy. Use this to confirm valid filter values before querying — e.g. which Versions or Periods exist.

Query Tools

query_cube

Query a cube via a named public view. Supports pandas-style filter_expr (e.g. Version == 'Actual' and Value > 1000) and column sorting. Best for structured exploration without writing MDX.

cube_nameview_namefilter_exprsort_bysort_ordertop_n

execute_mdx

Run a native TM1 MDX SELECT statement for precise cross-dimensional queries. Use when view-based filtering can't express the slice you need — e.g. CROSSJOIN of versions and periods.

mdxtop_nskip_zeros
